🌐 Que fait un serveur web lorsqu’il reçoit une requête HTTP ?

Lorsqu’un serveur web reçoit une requête HTTP, il suit plusieurs étapes :


1. 📡 Réception de la requête
Le serveur :
	écoute sur un port (souvent 80 pour HTTP, 443 pour HTTPS)
	reçoit la requête envoyée par le client (navigateur)


2. 🔍 Analyse de la requête
Le serveur lit les éléments de la requête :
	méthode (GET, POST, etc.)
	URL (ressource demandée)
	headers
	body (si présent)

👉 Cela lui permet de comprendre ce que veut le client


3. 🧭 Routage de la requête
Le serveur décide quoi faire en fonction de l’URL :
	/index.html → fichier statique
	/api/users → appel à une API
	/login → traitement d’un formulaire

👉 Cette étape s’appelle souvent le routing


4. ⚙️ Traitement
Selon la requête, le serveur peut :
	🔹 Servir un fichier statique
	HTML, CSS, image…
	🔹 Exécuter du code
	PHP, Node.js, Python, etc.
	🔹 Accéder à une base de données
	récupérer ou stocker des informations
	
	
5. 🧠 Génération de la réponse
Le serveur construit une réponse HTTP :
	code de statut (ex : 200, 404, 500)
	headers (type de contenu, cache…)
	body (HTML, JSON, etc.)


6. 📤 Envoi de la réponse
Le serveur renvoie la réponse au client.

👉 Le navigateur :
	reçoit les données
	les interprète
	affiche le résultat


7. 🔚 Fin de la connexion
En HTTP/1.0 → connexion fermée
En HTTP/1.1 → souvent gardée ouverte (keep-alive)

🔁 Résumé simple
👉 Cycle complet :
	Le client envoie une requête
	Le serveur analyse
	Il traite la demande
	Il renvoie une réponse
	Le client affiche


Index && Auto Index

🔹 index — fichier par défaut

La directive index sert à dire à NGINX :

👉 “Quel fichier je dois servir quand on accède à un dossier ?”

Exemple
location / {
    index index.html index.htm;
}

👉 Si quelqu’un va sur :

/blog/

NGINX va chercher dans le dossier :

index.html
sinon index.htm

✔️ S’il en trouve un → il le renvoie
❌ Sinon → erreur (souvent 403 ou 404)

🔹 autoindex — affichage du dossier

La directive autoindex permet d’afficher la liste des fichiers d’un dossier.

Exemple
location /files/ {
    autoindex on;
}

👉 Si quelqu’un va sur :

/files/

✔️ NGINX affiche une page avec :

liste des fichiers
tailles
dates
⚠️ Interaction entre les deux

C’est là que ça devient intéressant :

Cas 1 — index trouve un fichier
index index.html;
autoindex on;

👉 /docs/ contient index.html
➡️ NGINX sert index.html
➡️ autoindex n’est pas utilisé

Cas 2 — pas de fichier index

👉 /docs/ ne contient pas index.html

autoindex off → ❌ erreur
autoindex on → ✔️ liste des fichiers
🔹 Résumé simple
Directive	Rôle
index	fichier à servir par défaut
autoindex	afficher le contenu du dossier
